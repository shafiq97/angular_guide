Introduction

Before setting up Angular, ensure that you have the following installed on your system:

  • Node.js and npm: Angular requires Node.js and its package manager, npm.
  • Angular CLI: Command Line Interface tool to initialize, develop, scaffold, and maintain Angular applications.
  • Code Editor: Such as Visual Studio Code.

Setting Up Angular on Mac

1. Install Homebrew (Optional but Recommended)

Homebrew is a popular package manager for macOS. It simplifies the installation of software.

  1. Open the Terminal.
  2. Install Homebrew by running: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  3. After installation, ensure Homebrew is up to date: 
    brew update

2. Install Node.js and npm

You can install Node.js either via Homebrew or by downloading the installer.

  1. Using Homebrew: 
    brew install node
  2. Using Installer:
    • Go to the Node.js official website.
    • Download the LTS version for macOS.
    • Run the installer and follow the prompts.

Verify the installation by running:

node -vnpm -v

3. Install Angular CLI

  1. In the Terminal, run the following command to install Angular CLI globally: 
    npm install -g @angular/cli
  2. Verify the installation by checking the Angular CLI version: 
    ng version
Note: If you encounter permission issues during installation, you might need to use sudo. However, it's generally recommended to configure npm to install packages without requiring sudo.

Troubleshooting

Here are some common issues you might encounter during the setup and how to resolve them:

1. Permission Errors

Solution: Avoid using sudo with npm. Instead, configure npm to install global packages without requiring elevated privileges.
  1. Create a directory for global installations: 
    mkdir ~/.npm-global
  2. Configure npm to use the new directory path: 
    npm config set prefix '~/.npm-global'
  3. Add the new directory to your PATH in .bashrc, .bash_profile, or .zshrc: 
    export PATH=~/.npm-global/bin:$PATH
  4. Apply the changes: 
    source ~/.bashrc
    or 
    source ~/.zshrc

2. Angular CLI Not Recognized

If you receive a message like 'ng' is not recognized, it means that Angular CLI is not installed globally or not added to your PATH.

Solution:
  1. Ensure Angular CLI is installed globally: 
    npm install -g @angular/cli
  2. Verify the installation: 
    ng version
  3. If the issue persists, check your PATH environment variable to ensure npm's global bin directory is included.

3. Port 4200 Already in Use

If you try to run ng serve and receive an error that port 4200 is already in use, another application is occupying the port.

Solution:
  1. Find the process using port 4200:
    • Windows: 
      netstat -aon | findstr :4200
      Note the PID (Process ID) and terminate it using: 
      taskkill /PID  /F
    • Mac: 
      lsof -i :4200
      Note the PID and terminate it using: 
      kill -9
  2. Alternatively, serve the application on a different port: 
    ng serve --port 4300
